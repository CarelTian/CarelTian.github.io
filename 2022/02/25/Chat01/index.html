开端

关于我为什么要做一个博客，只能说一时兴起。我其实很早以前就铁了心的不想做web开发，因为单纯的不喜欢。相比较而言，更偏向于C/S结构的软件。后来经过慢慢地摸索，觉得web也挺有意思的。因为不用造轮子，有大量的框架，模板可以利用。最近有个想法，想要开始写日志。把日志在静态网页上优点有很多，比如

  1. 美观。把想法保留在电脑上很简单，创建一个记事本或Word文档都可以，只是写起来枯燥乏味以至于写完之后也不会看上一眼
  2. 成就感。尽管使用了很多别人造的轮子，站点的维护也需要大量工作。

废话就不说了，开始搭建。

过程

如果使用了别人做的主题，那就必须要查看文档。文档中包含了很多配置项，能够指导你个性化使用主题，发布文章的操作流程。如何配置这里不多说，不同主题的使用方法也是不同。配置好后，使用hexo -s 命令预览效果，如果ok那就开始部署。

1.部署在服务器中

下载git包

apt-get update
apt-get install git -y

创建仓库文件夹,并修改读写权限

mkdir /var/repo/
chown -R $USER:$USER /var/repo/
chmod -R 755 /var/repo/

创建远程仓库

cd /var/repo
git init --bare myblog.git

创建git钩子（仓库与服务器根目录同步）

vim /var/repo/myblog.git/hooks/post-receive
git --work-tree={Apache服务器根目录} --git-dir=/var/repo/myblog.git checkout -f

增加可执行权限

chmod +x /var/repo/myblog.git/hooks/post-receive

回到本地操作系统上，在hexo站点配置_config.yml。更改其中的url和deploy，马赛克处是ip地址


大功高成，还差最后一步命令上传到git仓库

hexo clean & hexo g -d
